int  numberOfShades

Method to convert a BGR image into a grayscale image.

This algorithm allows the user to choose how many shades of gray the resulting image will be. details Any value between 2 and 256 is accepted. (the value is entered as a parameter -> numberOfShades)

2 represents a black and white image, 256 represents an identical image as the Averaging algorithm.

This method uses the next formulas:

ConversionFactor = 256 / numberOfShades

AverageValue = (Red + Green + Blue) / 3

Gray = Integer((AverageValue / ConversionFactor) + 0.5) * ConversionFactor

The algorithm takes a color and converts it to its least saturated variant. details A pixel can be desaturated by finding the middle between the maximum (B, G, R) and the minimum (B, G, R).

This method uses the formula: Gray = ( Max(Red, Green, Blue) + Min(Red, Green, Blue) ) / 2

bool ALGORITHMS_API Algo::Luminance ( const cv::Mat &  inImage,
cv::Mat &  outImage 
)

Method to convert a BGR image into a grayscale image. (The weighted method)

Used in color television and video systems such as PAL, SECAM and NTSC.

Use weights 0.3 for red, 0.59 for green and 0.11 for blue.

The color red is the most bright in an image and the color green gives a calming effect for eyes.

Therefore, according to the standard ITU-R Rec. 601 or BT.601, the red color contribution is reduced and

the green one is increased, the contribution of the blue color being between these two.

This method uses the formula: Gray = (Red * 0.3 + Green * 0.59 + Blue * 0.11)
